Array Is a Hash? are arrays in this language really 'associative arrays', that is, the same as hashes?

Yes, for the most part. Arrays can store values under character string and other kinds of keys, just like any other object. But the array methods only work on integers 0 (zero) and above. 

animals.toString()
= 'dog,pig,chicken'
animals['pet'] = 'turtle';
animals[-55] = 'spider';
animals.toString()
= 'dog,pig,chicken'
animals.length
= 3
for (k in animals) console.log(k);
=> 0
=> 1
=> 2
=> pet
=> -55

Yes, an 'associative array'. Keys can either be integers or strings.

PHP arrays remember their element's assigned order. Some operations use the assigned order of elements rather than the indices; see #Shift and #Unshift below for an example.

See Hashes for more info.

No, unrelated.

Array Is an Object? are arrays a class of Object? or are they outside of the class system?

Arrays are instances of a class Array, which adds a few methods and properties to a plain object.

No, unrelated.

An Array is an instance of class Array, but elements of an Array are unrelated to members of the Array object.

Concatenate modify array, or construct new array, to include the elements of another array on its end

returns new array: 

animals + ['bear', 'ostrich']
= ['dog', 'pig', 'chicken', 'bear', 'ostrich']

modifies: 

animals += ['bear', 'ostrich']
animals= ['dog', 'pig', 'chicken', 'bear', 'ostrich']

the concat() function behaves just like + and allows multiple arguments: 

animals.concat(['bear', 'ostrich'], ['fish'])
= ['dog', 'pig', 'chicken', 'bear', 'ostrich', 'fish']

array_splice($animals, count($animals), 0, array('bear', 'ostrich'));
$animals= array('dog', 'pig', 'chicken', 'bear', 'ostrich')

array_merge($animals, array('bear', 'ostrich'));
$animals= array('dog', 'pig', 'chicken', 'bear', 'ostrich')

array_merge() works differently with string arguments

In both cases above, existing keys are ignored; order comes implied in the array. Resulting keys are renumbered starting with 0. 

$animals = array_merge(array(88=>'bear', 4=>'ostrich'), $animals);
$animals= array('bear', 'ostrich', 'dog', 'pig', 'chicken', )

Iterate loop that visits each element of the array
for (var i = 0; i < animals.length; ++i)
    console.log("Where is that "+ animals[i] +"? ");
=> 'Where is that dog? Where is that pig? Where is that chicken? '

Packages like Prototype have methods to do this more gracefully. 

for (k in animals)
    console.log("Where is that "+ animals[i] +"? ");
=> 'Where is that dog? Where is that pig? Where is that chicken? '

That's actually the hashtable way to iterate through all entries; it'll also include variables and methods that are attached to this object (excluding those in the next prototype). 

foreach ($animals as $ani)
    print("Where is that $ani? ");
=> Where is that dog? Where is that pig? Where is that chicken?

This goes in implied order, not key order.

or with the index: 

foreach ($animals as $ix => $ani)
    print("($ix) Where is that $ani? ");
=> (0) Where is that dog? (1) Where is that pig? (2) Where is that chicken?

To iterate with a callback function, use array_walk().

PHP arrays also contain an implied pointer that's more flexible but more dangerous. Mostly you'll find these in older code. The main problem is that the loops usually end if they encounter a false value like 0, null or an empty string. foreach() uses this pointer, but safely. Mostly these loops looked like this: 

for ($ani = reset($animals); $ani; $ani = next($animals))
	print("$ani ");
=> dog pig chicken

They still work OK if no value encountered is false, but their use is discouraged.

by value: 

animals.each {|ani| print("Where is that #{ani}? ") }
=> Where is that dog? Where is that pig? Where is that chicken?

by index: 

animals.each_index {|ix| print("Where is that #{animals[ix]}? ") }
=> Where is that dog? Where is that pig? Where is that chicken?

both: 

animals.each_with_index {|ani, ix| print("(#{ix}) Where is that #{ani}? ") }

Filter reconstruct array, including only those elements approved by a function you write

Sorry, must be done manually. Packages like Prototype have ways to do this better. 

filterProc = function(ani) {return ani.index('g') >= 0};
newan = [];
for (var i = 0; i < animals.length; ++i)
    if (filterProc(animals[i]))
        newan.push(animals[i]);
["dog", "pig"]

Construct an array with just the animals that contain the letter 'g': 

function hasG($ani) {return strpos($ani, 'g') !== false;}
array_filter($animals, 'hasG')
= array('dog', 'pig')

just the animals that contain the letter 'g': 

animals.select {|ani| ani['g'] }
=  ["dog", "pig"]

invert the sense 

animals.reject {|ani| !ani['g'] }
=  ["dog", "pig"]

select() and reject() also work on any Enumerable, producing an array, like this: 

(1...100).select {|n| n % 13 == 0 }
= [13, 26, 39, 52, 65, 78, 91]
